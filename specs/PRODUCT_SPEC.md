# 产品规格：2026 世界杯 AI 预测助手

## 1. 产品定位

这是一个本地运行的 2026 世界杯预测与赛程信息网页应用，面向没有软件工程背景的普通用户。用户通过浏览器完成查看分组、浏览赛程、查看已完赛结果、关注场外资讯、生成预测、配置 LLM 和导出报告，不需要接触 Python 命令、CSV 字段或模型公式。

产品采用双模式定位：

- 基础模式：用户未配置 LLM 时，系统根据完整国际比赛历史数据、正式分组、公开可信赛程/赛果数据、已录入比分和本地统计模型生成预测，并提供近期赛程和已完赛结果浏览。
- AI 增强模式：用户配置 LLM 后，系统支持球队介绍、比赛进程更新、场外资讯关注、AI 信息因子设置、影响因素解释和预测报告生成。

产品的预测能力由两部分组成：

- 本地统计模型：负责计算球队晋级与夺冠概率。
- LLM：负责阅读用户指定的信息源，生成球队介绍，辅助更新比赛进程，提取影响因素，解释模型结果，并生成面向普通读者的报告。

## 2. 目标用户

主要用户：

- 对世界杯感兴趣的普通球迷。
- 想把 AI 预测过程写成文章、视频脚本或社交媒体内容的创作者。
- 懂一点数据但不想写代码的分析爱好者。

非目标用户：

- 专业博彩交易员。
- 需要实时赔率套利的用户。
- 需要企业级多人协作和权限管理的团队。

## 3. 用户问题

用户想知道：

- 哪些球队最可能夺冠？
- 最近几天有哪些比赛？
- 已结束比赛的比分是多少？
- 已结束比赛之后，预测发生了什么变化？
- 某场冷门为什么影响了晋级概率？
- 天气、伤病、球队新闻和场外资讯是否会改变预测？
- AI 的判断依据是什么，哪些地方不确定？

当前脚本的问题：

- 需要命令行。
- 字段命名偏工程化。
- 没有运行时 AI 参与。
- 用户无法方便录入已结束比分。
- 外部信息源不能进入预测解释流程。

## 4. MVP 范围

MVP 必须包含：

- 本地网页界面。
- OpenAI 兼容 LLM API 配置。
- 2026 世界杯正式分组内置。
- 已结束比赛比分录入。
- 后续需要支持 AI 辅助录入比分：用户粘贴赛果文本或提供权威赛果 URL，LLM 提取待确认比分，用户确认后写入。
- 可视化预测结果。
- 用户可添加权威信息源 URL。
- LLM 对信息源做摘要和结构化影响因子提取。
- 本地模型基于比分和影响因子重新模拟。
- LLM 生成普通读者能看懂的预测报告。
- 所有数据保存在本地。
- 可打包成压缩包交付。

MVP 不包含：

- 自动全网搜索。
- 自动登录付费网站。
- 实时比分推送。
- 多用户账号系统。
- 云端部署。
- 博彩下注建议。
- 球员级别完整数据库。

## 5. 核心流程

1. 用户打开本地网页。
2. 系统提示用户配置 LLM API。
3. 用户进入赛事面板，查看默认正式分组。
4. 用户录入已结束比赛比分。
5. 后续版本中，用户可粘贴赛果文本或输入权威赛果 URL，让 LLM 生成待确认比分列表。
6. 用户添加一个或多个权威信息源 URL，并标注用途，例如天气、伤病、赛前报道。
7. 系统调用 LLM，把信息源内容转成结构化影响因子。
8. 系统运行本地预测模型。
9. 系统展示图表、概率变化和 AI 解读。
10. 用户导出 HTML 报告。

## 6. AI 参与边界

AI 应该做：

- 解释用户输入。
- 从用户提供的赛果文本或权威赛果页面中提取待确认比分。
- 总结权威来源内容。
- 抽取结构化影响因子。
- 指出预测的不确定性。
- 把模型结果写成自然语言报告。

AI 不应该做：

- 直接决定冠军。
- 在无来源时编造伤病、天气、阵容信息。
- 覆盖本地模型的概率计算。
- 给出博彩下注建议。
- 隐藏模型和数据局限。

## 7. 成功标准

MVP 成功的标准：

- 用户不看 README 也能通过页面完成一次预测。
- 用户能理解图表上的每个指标。
- 用户能录入比分并看到概率更新。
- 用户能配置任意 OpenAI 兼容 API 服务。
- 如果 LLM 请求失败，基础预测仍能运行。
- 生成的报告说明了数据来源、影响因素和不确定性。

## 8. 风险

- 权威信息源页面结构复杂，抓取失败。
- LLM 可能提取错误或过度解读。
- 普通用户可能误把预测当成确定结论。
- 本地环境差异导致启动失败。
- 完整历史比赛数据下载可能被网络环境阻断。

对应策略：

- MVP 允许用户粘贴网页正文作为备选。
- LLM 输出必须带置信度和来源引用。
- 页面显著展示“概率不是确定结果”。
- 提供一键启动脚本和清晰错误提示。
- 内置样例数据，真实数据可手动导入。

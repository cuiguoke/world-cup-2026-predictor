# 2026 世界杯 AI 预测产品 MVP 规格

本目录是按 SDD（Spec-Driven Development，规格驱动开发）推进的 MVP 规格版。

目标不是立刻堆功能，而是先把产品边界、用户体验、AI 参与方式、数据结构和实现计划写清楚。后续实现时，每一项开发都应能追溯到这里的规格。

## 文档清单

- `PRODUCT_SPEC.md`：产品定位、目标用户、MVP 范围、成功标准。
- `UX_SPEC.md`：本地网页的信息架构、页面流程、视觉与交互原则。
- `AI_SPEC.md`：LLM 参与边界、输入输出协议、失败兜底、安全策略。
- `DATA_SPEC.md`：比赛、比分、信息源、AI 影响因子、预测结果的数据结构。
- `IMPLEMENTATION_PLAN.md`：MVP 技术方案、开发阶段、验收标准。

## MVP 一句话

一个给普通球迷使用的本地网页应用：用户配置 OpenAI 兼容 LLM API、录入已结束比分、添加权威信息源，系统用本地统计模型计算概率，并用 LLM 提取外部因素、解释变化、生成普通人能看懂的预测报告。

## 核心原则

1. 预测计算在本地完成，LLM 不直接“拍脑袋猜冠军”。
2. AI 参与信息整理、因素提取、解释与质疑。
3. 普通用户界面隐藏公式、字段缩写和文件格式。
4. 高级参数可调整，但默认路径必须足够简单。
5. 所有预测结果都要明确标注不确定性和数据来源。

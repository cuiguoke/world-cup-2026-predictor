# UX 规格：本地网页体验

## 1. 设计目标

界面应当让普通用户感觉这是一个“世界杯预测工作台”，不是代码项目。视觉风格简洁、现代、克制，重点突出预测结果、已结束比赛和 AI 解读。

## 2. 视觉方向

- 背景：浅灰白或柔和冷色背景。
- 主色：绿色/蓝绿色，呼应球场与数据可信感。
- 辅色：深墨绿、灰蓝、少量琥珀色用于提醒。
- 字体：系统无衬线字体。
- 卡片：只用于独立信息块，圆角不超过 8px。
- 图表：清晰、留白充足，避免复杂装饰。

避免：

- 夸张渐变背景。
- 大面积紫色科技感。
- 营销式首屏。
- 把公式、JSON、CSV 字段暴露给普通用户。

## 3. 信息架构

MVP 页面包含 5 个主要区域：

1. 顶部状态栏
2. 设置向导
3. 比分录入
4. 预测结果
5. AI 分析报告

### 3.1 顶部状态栏

显示：

- 当前赛事：2026 世界杯
- 数据状态：样例数据/完整历史数据
- LLM 状态：未配置/已连接/请求失败
- 最近一次预测时间

### 3.2 设置向导

用于首次启动。

步骤：

1. 选择数据来源
   - 使用内置样例数据
   - 导入完整历史比赛数据
2. 配置 LLM API
   - Base URL
   - API Key
   - 模型名
   - 测试连接按钮
3. 确认分组
   - 默认使用正式分组
   - 允许编辑球队名称

### 3.3 比分录入

用户按小组和日期录入已结束比赛。

每场比赛字段：

- 主队
- 客队
- 主队进球
- 客队进球
- 状态：未开始/已结束

交互：

- 输入比分后自动标记为已结束。
- 支持撤销比分。
- 支持批量保存。
- 保存后显示“预测需要重新生成”。

### 3.4 预测结果

默认展示：

- 夺冠概率 Top 10
- 进入决赛概率
- 进入四强概率
- 分组出线概率
- 与上次预测相比的变化

普通用户标签：

- 夺冠
- 进决赛
- 进四强
- 进八强
- 进16强
- 进32强

高级字段隐藏在“查看模型细节”中。

### 3.5 AI 分析报告

报告结构：

- 最可能夺冠球队
- 概率变化最大的球队
- 已结束比赛造成的影响
- 外部信息源提取到的关键因素
- 模型最不确定的地方
- 需要用户谨慎看待的结论

## 4. 关键用户任务

### 任务 A：第一次运行

1. 打开本地网页。
2. 看到“开始配置”。
3. 填入 LLM API。
4. 点击测试连接。
5. 选择使用内置数据或导入数据。
6. 点击“生成第一次预测”。

验收：

- 用户不需要打开终端。
- 页面能解释每一步为什么需要。

### 任务 B：更新已结束比赛

1. 进入“比分录入”。
2. 找到某场比赛。
3. 输入比分。
4. 点击“重新预测”。
5. 查看概率变化。

验收：

- 用户能看出哪支球队概率上升或下降。
- 页面解释变化不是确定因果，而是模型结果。

### 任务 C：添加权威信息源

1. 进入“AI 信息源”。
2. 输入 URL。
3. 选择类型：天气/伤病/阵容/赛前报道/其他。
4. 点击“提取影响因素”。
5. 审核 LLM 提取结果。
6. 勾选是否纳入预测解释。

验收：

- LLM 提取结果必须让用户确认。
- 未确认的信息不直接影响报告结论。

## 5. 状态与错误

必须处理：

- LLM 未配置。
- LLM 连接失败。
- URL 抓取失败。
- LLM 输出格式不合法。
- 比分输入非法。
- 数据文件缺失。
- 预测运行中。
- 预测完成但有警告。

错误提示风格：

- 用普通语言解释。
- 给出下一步操作。
- 不展示 Python traceback 给普通用户。

## 6. MVP 页面清单

MVP 可以做成单页应用，使用顶部标签切换：

- `预测总览`
- `比分录入`
- `AI 信息源`
- `设置`
- `导出报告`

每个标签都应可独立刷新，不丢失本地保存的数据。
